# Context Explorer Workflow Guide
## PRISM Project Development Intelligence & Session Management

---

## Overview

The Context Explorer is a real-time development intelligence platform for the PRISM project, providing comprehensive visibility into session progress, code metrics, and development patterns across all 18 modules.

**Current Status**: Operational at 6,389 lines of code across 4 development sessions  
**URL**: http://localhost:3003/ (development) | TBD (production)  
**Integration**: Automated session wrap parsing and metrics aggregation

## 1. CONTEXT EXPLORER CAPABILITIES

### Real-Time Development Metrics
- **Total Lines of Code**: Live aggregation across all modules (currently 6,389)
- **Session Timeline**: Chronological development history with achievements
- **Module Progress**: Individual module completion status and complexity
- **Integration Patterns**: Cross-module dependencies and data flow

### Session Intelligence Features
- **Achievement Tracking**: Automated extraction from session wraps
- **Decision Documentation**: Technical decisions with rationale and alternatives
- **Challenge Resolution**: Problem-solving patterns and solutions
- **Performance Metrics**: Response times, test coverage, quality scores

### Development Analytics
- **Complexity Analysis**: Code complexity trends and patterns
- **Integration Success**: Module interconnectivity validation
- **Quality Benchmarks**: TypeScript compliance, test coverage, regulatory accuracy
- **Timeline Visualization**: Development velocity and milestone tracking

## 2. UPDATED DEVELOPMENT WORKFLOW

### Phase 1: Pre-Session Setup with Context Explorer

#### Session Planning Process
```yaml
1. Context Explorer Review:
   - Launch: http://localhost:3003/
   - Review timeline page for development history
   - Analyze previous session achievements and challenges
   - Identify integration dependencies and patterns

2. Module Preparation:
   - Review specific module requirements from master context
   - Check HTML prototype source files
   - Verify regulatory compliance requirements
   - Plan integration touchpoints with completed modules

3. Success Criteria Definition:
   - Define specific deliverables based on Context Explorer insights
   - Set realistic complexity targets based on previous sessions
   - Plan testing strategy using established patterns
   - Identify integration validation requirements
```

#### Context File Updates
```yaml
Required Updates:
   - Update master_context_PRISM.md with latest completion status
   - Create/update module-specific .claude.md files
   - Review integration patterns from successful sessions
   - Validate regulatory compliance requirements
```

### Phase 2: Active Development with Context Explorer Integration

#### Real-Time Development Monitoring
```yaml
Development Environment:
   - Keep Context Explorer open in browser during development
   - Monitor metrics dashboard for real-time feedback
   - Reference previous session patterns for technical decisions
   - Track lines of code growth against complexity targets

Session Documentation:
   - Follow session_wrap_PRISM.md template consistently
   - Document all technical decisions with regulatory context
   - Include accurate lines of code counts using actual measurements
   - Reference Context Explorer insights for pattern recognition
```

#### Integration Validation Process
```yaml
Continuous Integration:
   - Test with existing modules (Firm Data, Financial Data, FOR Calculator)
   - Validate data flow patterns established in previous sessions
   - Ensure TypeScript compliance with existing architecture
   - Verify regulatory calculation accuracy against HTML prototypes

Quality Assurance:
   - Maintain Context Explorer metrics standards
   - Follow established testing patterns from successful sessions
   - Document performance benchmarks and optimization decisions
   - Ensure audit trail completeness for regulatory compliance
```

### Phase 3: Post-Session Context Management

#### Context Explorer Update Process
```yaml
Immediate Actions:
   1. Session Wrap Completion:
      - Complete session wrap following established template
      - Include accurate lines of code measurements
      - Document all decisions, challenges, and achievements
      - Ensure regulatory compliance documentation

   2. Context Explorer Refresh:
      - Run: node scripts/copySessionWraps.js
      - Verify new session appears in timeline
      - Validate metrics aggregation accuracy
      - Check total lines of code calculation

   3. Integration Validation:
      - Test Context Explorer with new session data
      - Verify parsing accuracy for new session wrap
      - Confirm metrics calculations are correct
      - Validate timeline chronology
```

#### Context File Maintenance
```yaml
Strategic Updates:
   - Update master_context_PRISM.md completion percentages
   - Document new integration patterns discovered
   - Update module inventory with latest status
   - Validate regulatory compliance achievements

Pattern Documentation:
   - Document successful development patterns
   - Record integration solutions and approaches
   - Update architectural decisions and rationale
   - Maintain regulatory compliance evidence
```

## 7. TROUBLESHOOTING & SUPPORT

### Common Issues and Solutions
```yaml
Context Explorer Not Loading:
   - Check development server status (npm run dev)
   - Verify session wrap files are properly copied
   - Confirm parser functionality with test scripts
   - Check browser console for error messages

Metrics Calculation Errors:
   - Verify session wrap format consistency
   - Check lines of code parsing patterns
   - Validate number formatting (commas, decimals)
   - Confirm aggregation logic accuracy

Session Wrap Parsing Issues:
   - Validate session wrap template compliance
   - Check for consistent section headers
   - Verify metadata format accuracy
   - Confirm file encoding and accessibility
```
